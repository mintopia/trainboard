# M4: OTG Gadget Ethernet + mDNS Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** USB gadget ethernet as an app-independent debug lifeline (plug into any computer → `http://10.55.0.1`) + stdlib mDNS so the board answers to `trainboard-XXXX.local` / `trainboard.local` on every LAN-ish interface.

**Architecture:** Gadget = declarative OS config in `deploy/gadget/` (configfs script + oneshot unit + usb0-scoped dnsmasq) — no Go owns USB descriptors; `internal/net` untouched. App side = one new package `internal/mdns` (announce-only responder, hand-rolled wire format for exactly A/AAAA/PTR/SRV/TXT), wired from cmd in both boot paths with a hotspot-suppression seam. Spec: `docs/superpowers/specs/2026-07-08-m4-otg-mdns-design.md`.

**Tech Stack:** Go stdlib only. Bash (shellcheck-clean) for the gadget script.

## Global Constraints

- Branch `feat/m4-otg-mdns`. TDD red→green per task; `make check` exit 0 before any push.
- `internal/mdns` must import NEITHER `internal/net` NOR `internal/web`; cross-talk only via funcs wired in `cmd/trainboard` (project idiom).
- No new Go dependencies (stdlib only — repo currently has argon2/x-crypto only).
- usb0 literals: Pi `10.55.0.1/29`, DHCP range `10.55.0.2–10.55.0.6`, 12h leases, NO router/DNS options pushed.
- Gadget: NCM function preferred + ECM legacy, NEVER RNDIS. VID/PID `0x1d6b:0x0104`. MACs derived from device-tree serial (stable across boots). Teardown must be complete and reversible.
- mDNS names: `trainboard-XXXX.local` (XXXX = existing `macTail` result, uppercase hex) + `trainboard.local` alias + `_http._tcp.local` service (instance `Trainboard XXXX`, port 80, TXT `path=/`). Announce-only: no probing, no conflict rename. Goodbye (TTL 0) on shutdown.
- Interface eligibility: up ∧ multicast ∧ has usable unicast addr ∧ NOT (wlan0 while hotspot active). Churn poll every 5s.
- No new HTTP routes (e2e route matrix must be untouched).

### Task 1: mDNS wire format (marshal/unmarshal)

**Files:**
- Create: `internal/mdns/wire.go`, `internal/mdns/wire_test.go`

**Interfaces (produced, consumed by Tasks 2/3):**

```go
package mdns

// Uncompressed DNS wire format for the exact records this responder emits.
// Decode tolerates name compression (queriers use it); encode never emits it.

type header struct {
	ID      uint16
	Flags   uint16 // response bit 0x8000 | authoritative 0x0400 for answers
	QDCount, ANCount, NSCount, ARCount uint16
}

type question struct {
	Name  string // fully qualified, lowercase, trailing dot stripped: "trainboard-ab12.local"
	Type  uint16 // TypeA etc.
	Class uint16 // 0x0001 (IN); top bit (unicast-response) masked off by parser
}

type record struct {
	Name  string
	Type  uint16 // TypeA=1, TypeAAAA=28, TypePTR=12, TypeSRV=33, TypeTXT=16
	Class uint16 // 0x0001, or 0x8001 (cache-flush) for our authoritative answers
	TTL   uint32 // 120 for A/AAAA, 4500 for PTR/SRV/TXT, 0 for goodbye
	Data  rdata
}

type rdata interface{ encode(buf *bytes.Buffer) error }
type rdataA struct{ IP net.IP }       // 4 bytes
type rdataAAAA struct{ IP net.IP }    // 16 bytes
type rdataPTR struct{ Target string } // name
type rdataSRV struct{ Priority, Weight, Port uint16; Target string }
type rdataTXT struct{ Pairs []string } // length-prefixed strings

func encodeMessage(h header, qs []question, ans []record) ([]byte, error)
func decodeQuery(pkt []byte) (header, []question, error) // answers in queries are ignored (known-answer suppression NOT implemented — announce-only responder, acceptable)
```

### Task 2: record set + answer logic

**Files:**
- Create: `internal/mdns/records.go`, `internal/mdns/records_test.go`

**Interfaces (produced, consumed by Task 3):**

```go
// Zone is the immutable set of names/records this responder serves for ONE
// interface snapshot (addresses baked in at construction).
type Zone struct { /* unexported fields */ }

// NewZone builds the zone. suffix = macTail output ("AB12"); host names are
// lowercased: "trainboard-ab12.local" + "trainboard.local".
// v4/v6 = the interface's usable unicast addresses (either may be nil).
func NewZone(suffix string, v4, v6 net.IP) *Zone

// Answers returns authoritative answer records for a question, nil if the
// question isn't ours. Matching is case-insensitive per RFC 6762 §16.
// Handles: A/AAAA for both hostnames; PTR for _http._tcp.local;
// SRV/TXT for the service instance; PTR for _services._dns-sd._udp.local
// (service-type enumeration → _http._tcp.local).
func (z *Zone) Answers(q question) []record

// Announcement returns the full unsolicited announce set (all records,
// cache-flush class on A/AAAA/SRV/TXT, plain class on the two PTRs).
func (z *Zone) Announcement() []record

// Goodbye returns Announcement() with every TTL forced to 0.
func (z *Zone) Goodbye() []record
```

### Task 3: responder loop

**Files:**
- Create: `internal/mdns/responder.go`, `internal/mdns/responder_test.go`

**Interfaces (produced, consumed by Task 4):**

```go
// Config wires the responder. All fields required unless noted.
type Config struct {
	Suffix string // macTail result, e.g. "AB12"
	// SuppressWlan0 reports whether wlan0 must stay silent (hotspot up).
	// nil = never suppress.
	SuppressWlan0 func() bool
	// Beat is the watchdog heartbeat; nil = no-op. Called each poll tick.
	Beat func()
	Log  *slog.Logger
	// interfaces/conn seams — production defaults used when nil:
	Interfaces func() ([]net.Interface, error)          // default net.Interfaces
	ListenV4   func(ifi *net.Interface) (netPacketConn, error) // default real multicast join
	Addrs      func(ifi *net.Interface) ([]net.Addr, error)    // default ifi.Addrs
	Now        func() time.Time // default time.Now
}

// netPacketConn is the test seam over *net.UDPConn.
type netPacketConn interface {
	ReadFrom(b []byte) (int, net.Addr, error)
	WriteTo(b []byte, addr net.Addr) (int, error)
	Close() error
}

// Run blocks until ctx is done: polls interfaces every 5s (pollInterval
// const), opens/closes per-interface listeners on churn, answers queries,
// announces on interface-appear (twice, 1s apart, per RFC 6762 §8.3),
// sends goodbyes on ctx cancellation. Never returns a non-nil error except
// for setup impossibilities; per-packet errors are logged and survived.
func Run(ctx context.Context, cfg Config) error
```

Eligibility (unexported `eligible(ifi, addrs, suppressWlan0)`): `FlagUp` ∧ `FlagMulticast` ∧ ≥1 global unicast or private v4 addr ∧ !(name=="wlan0" && suppress()). Loopback excluded by FlagMulticast test on lo? NO — lo has multicast flag on some systems: exclude `FlagLoopback` explicitly.

- [ ] **Step 1: Failing tests** (fake PacketConns, channel-scripted):
  - query in → answer out with our A record, sent to `224.0.0.251:5353` (multicast reply; unicast-response bit NOT honored — announce-only simplification, note in doc comment);
  - non-matching query → no write;
  - interface appears between polls → listener opened + 2 announcements observed ~1s apart (fake clock);
  - wlan0 suppressed: SuppressWlan0 true → wlan0 never opened; flips false → opened + announced;
  - interface disappears → conn closed;
  - ctx cancel → goodbye (TTL 0) written on every open conn, Run returns nil;
  - malformed packet in → logged, loop survives (follow-up query still answered);
  - Beat called at least once per poll tick.
- [ ] **Step 2: Red.** **Step 3: Implement** (`pollInterval = 5 * time.Second` const; goroutine per conn reading into the shared handler; mutex-guarded conn map keyed by iface name+index). **Step 4: Green** — full `-race`. **Step 5: Commit** — `feat(mdns): announce-only responder with churn handling and AP suppression`
