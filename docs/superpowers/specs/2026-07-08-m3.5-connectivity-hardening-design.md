# M3.5: Connectivity Hardening — Design

**Issues:** #44, #45, #46, #47, #48, #50 (all filed at the 2026-07-08 bench/acceptance sessions). **Approved:** 2026-07-08 (Jess).

## Goal

Six surgical fixes to the now-hardware-validated connectivity stack. No
structural rework: the manager's phase/retry model survived the bench; every
bug here is parameter- or lifecycle-level. Approach chosen over a generic
"transition robustness layer" (rejected: churns validated code, YAGNI).

The usb0 gadget lifeline (M4) replaces dead-man ceremony for all on-hardware
verification in this milestone.

## 1. Manager fixes (`internal/net`)

### #47 — cold-start STA race (the 5-minute config-apply detour)

Observed 4× on hardware: the first STA attempt after the manager spawns its
own `wpa_supplicant -B` fails in <1s (ctrl socket not ready → first `wpa_cli`
call errors → attempt aborts pre-poll) → AP fallback → join succeeds only at
the +5 min retry.

Two independent fixes, both required:

- **ensureDaemon socket wait**: after spawning `wpa_supplicant -B`, poll
  `wpa_cli -i wlan0 status` up to 10×500ms (injected sleeper, same rhythm as
  `pollStatus`) until it answers; only then is the daemon "up". Timeout →
  error (existing failure path). No change when the daemon already runs.
- **Fast retry once**: in the manager's run loop, if the FIRST STA attempt
  of a process lifetime fails within 2s of starting (measured via the
  injected clock), immediately re-attempt ONCE before AP fallback. Bounded:
  one fast retry per process lifetime, never on later attempts (the periodic
  retry cadence owns those). Rationale: even with the socket wait, other
  cold-start races (driver init, rfkill settle) produce the same instant-fail
  signature; the fast retry is the cheap generic net.

### #48 — post-SIGKILL AP restore misses its budget (fatal → ~10 min watchdog recovery)

Observed on the destructive matrix row 4: after `pkill -9 wpa_supplicant`,
the next cycle's `toAP` → `ensureDaemon` restarts the daemon, but `StartAP`'s
10×500ms poll expires ("AP not active after 10 polls") → AP-restore failure
is fatal by design → manager exits → 150s watchdog reboot.

- **Longer budget after a daemon (re)start**: `ensureDaemon` reports whether
  it started the daemon; when it did, `StartAP`'s poll budget doubles to
  20×500ms (post-crash radio needs longer than 5s; clean-state bring-up
  stays at 10 polls — bench Exp 1 measured 1s).
- **One in-process AP-restore retry**: before declaring the fatal
  AP-restore failure, tear down and retry the full `toAP` ONCE. The
  "verified AP restore" invariant holds — verified on the retry. Second
  failure → fatal as today (watchdog backstop unchanged, ADR 0003
  discipline intact).

### #46 — DHCP lease renewal (decision: daemon-mode dhclient)

`staAttempt` currently runs `dhclient -1 -v wlan0` (one-shot; exits after
bind; nothing renews — hourly lease expiry on typical routers leaves a
window where the address may be reallocated).

- Drop `-1`: `dhclient -v -pf /run/trainboard-dhclient.pid wlan0` stays
  resident and renews. The driver owns the pidfile lifecycle exactly as it
  owns dnsmasq's: kill on STA teardown / AP transition / manager shutdown.
  Kill-before-start on every attempt (stale daemon from a previous attempt
  must never survive into a new one).
- The layered check is unchanged (it verifies the lease exists; the daemon
  keeps it alive).

## 2. Product fixes

### #44 — open (passwordless) setup AP (operator decision, risk accepted)

- mode2 conf AP block: `key_mgmt=NONE`, no `psk` (hostapd conf equivalent
  updated for parity even though unused).
- `board.Hotspot.Password` field removed; ripples: hotspot scene (glass
  shows SSID + URL only — golden regenerates), web seams/templates
  (`/setup` AP copy, status view), `config.GenerateAPPassword` +
  provisioning persistence deleted, `resolveAPPassword` in cmd deleted.
- `eval-mode2.sh --ap-pass` unchanged (bench tool exercising WPA2 remains a
  valid driver test).
- deploy.md + destructive-matrix rows mentioning the AP password updated.
- Only hotspot-scene goldens may change.

### #45 — Europe/London everywhere times are shown or compared

- New micro-package `internal/tz`: `func Location() *time.Location` —
  `time.LoadLocation("Europe/London")` with `import _ "time/tzdata"`
  fallback (binary +~450KB, accepted) so minimal images without zoneinfo
  still get it. Panics never: last-resort fallback UTC with a logged error.
- Threaded into: `internal/render/element_clock.go` (glass clock),
  `internal/config/powersaving.go` (window comparisons),
  `internal/data/timerecon.go` (Darwin HH:MM is UK wall-clock).
- Host TZ becomes irrelevant to the app. deploy.md note.
